# DSS Practice Lab

이 디렉터리는 `../IaC`의 `Vagrantfile`로 생성한 로컬 노드들을 바탕으로 `DSS(Distributed Storage System)`를 구축하고, 특정 노드 장애 상황에서도 시스템이 계속 `read/write`를 처리할 수 있는지 검증하기 위한 실습 공간이다.

- `../IaC`: 실습용 가상 노드 인프라 생성
- `./DSS`: 분산 저장소 구성 및 장애 허용 동작 검증

## 목적

이 실습의 핵심 목표는 단순히 여러 VM을 띄우는 것이 아니라, 분산 저장 시스템이 일부 노드 장애 상황에서도 서비스를 지속할 수 있는지 확인하는 것이다.

검증하려는 질문은 다음과 같다.

- 특정 노드가 죽어도 데이터를 읽을 수 있는가?
- 특정 노드가 죽어도 데이터를 쓸 수 있는가?
- 복제, quorum, failover 같은 메커니즘이 의도대로 동작하는가?
- 장애 복구 후 데이터 일관성과 재동기화가 유지되는가?

## 기반 인프라

`../IaC/Vagrantfile`은 `VirtualBox` 위에 `Ubuntu 24.04` 기반 노드 3개를 생성한다.

- `node1` - `192.168.56.11`
- `node2` - `192.168.56.12`
- `node3` - `192.168.56.13`

각 노드는 다음과 같이 초기 설정된다.

- `vagrant` 사용자로 SSH 접속 가능
- 노드 간 SSH 접속을 위한 `ed25519` 키 배포
- `node1`, `node2`, `node3` 호스트명 등록
- 실습에 필요한 기본 패키지 설치

따라서 현재 디렉터리는 이미 준비된 로컬 클러스터 위에서 DSS를 설치하고 동작을 검증하는 단계에 해당한다.

## 실습 범위

이 디렉터리에서 다루는 작업은 다음과 같다.

- DSS 설치 및 실행
- 클러스터 설정 파일 관리
- 데이터 복제 또는 분산 정책 적용
- 장애 상황 주입
- 장애 상황에서의 `read/write` 테스트
- 복구 이후 상태 및 데이터 일관성 확인

## 기본 진행 흐름

1. `../IaC`에서 Vagrant 기반 노드들을 생성한다.
2. `node1`~`node3`에 분산 저장 시스템을 설치하고 클러스터를 구성한다.
3. 정상 상태에서 `read/write`가 수행되는지 먼저 확인한다.
4. 특정 노드를 중단하거나 죽인 상태를 만든다.
5. 남은 노드만으로도 `read/write`가 계속 가능한지 검증한다.
6. 장애 노드를 복구한 뒤 재동기화와 데이터 일관성을 확인한다.

## 장애 검증 시나리오

실습에서는 보통 다음과 같은 흐름으로 확인한다.

### 1. 정상 상태 확인

- 클러스터 전체 노드가 정상적으로 동작하는지 확인
- 테스트 데이터를 기록하고 읽어 정상 동작 여부 확인

### 2. 노드 장애 유도

- `node1` 또는 `node2` 또는 `node3` 중 하나를 중단
- 프로세스 종료, VM 중지, 네트워크 차단 등으로 장애 상황 재현

### 3. 장애 상황에서의 동작 확인

- 남은 노드만으로 읽기 가능한지 확인
- 남은 노드만으로 쓰기 가능한지 확인
- 장애 감지 및 리더 선출 또는 quorum 유지 여부 확인

### 4. 복구 확인

- 중단했던 노드를 다시 복구
- 누락 데이터 동기화 여부 확인
- 전체 클러스터 상태가 다시 정상으로 돌아오는지 확인
